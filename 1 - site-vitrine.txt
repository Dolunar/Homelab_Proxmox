# Guide complet — LXC Site sur Proxmox

------------------------------------------------------------------------------------------------
------------------------------------------------------------------------------------------------

différence entre LXC et VM :
Ce sont deux approches de virtualisation, mais à des niveaux différents.

VM (Machine Virtuelle)
Une VM émule un ordinateur complet, avec son propre noyau (kernel) et son propre matériel virtuel (CPU, RAM, disque, carte réseau...).
Repose sur un hyperviseur (VMware, VirtualBox, KVM, Hyper-V...)
Chaque VM a son propre OS complet (kernel inclus)
Isolation forte : une VM crashée n'affecte pas l'hôte
Lourd : démarre en secondes/minutes, consomme plus de RAM et disque


LXC (Linux Container)
Un LXC partage (utilise) le kernel de l'hôte et isole seulement l'espace utilisateur (processus, réseau, système de fichiers...).
Repose sur des fonctionnalités Linux : cgroups + namespaces
Isolation plus légère : si le kernel hôte crashe, tout tombe
Léger : démarre en millisecondes, overhead minimal

## Étape 2 — Créer le LXC

En haut à droite, clique sur **Create CT** puis remplis :

**Onglet General**
```
Hostname    : site
Password    : (choisis un mot de passe root solide)
```

**Onglet Template**
```
Template : debian-12-standard
```

**Onglet Disks**
```
Disk size : 8 GB  (largement suffisant)
```

**Onglet CPU**
```
Cores : 1
```

**Onglet Memory**
```
Memory : 512 MB
Swap   : 512 MB
```

**Onglet Network**
```
Bridge  : vmbr0
IPv4    : DHCP  (pour l'instant)
```

**Onglet DNS**
```
Laisser par défaut
```

Clique **Finish** et attends la création.

-------------------------------------------
Explication de Nginx : Nginx (prononcé "engine-x") est un serveur web open-source, créé en 2004 par Igor Sysoev.

il fait quoi ?
Il peut jouer plusieurs rôles selon la config :
1. Serveur web
Sert des fichiers statiques (HTML, CSS, images, JS) directement aux clients HTTP.

2. Reverse proxy
Se place devant une application (Node.js, Python, PHP...) et redirige les requêtes vers elle. C'est son usage le plus courant aujourd'hui.

3. Load balancer
Distribue le trafic entre plusieurs serveurs backend pour répartir la charge.

4. Cache HTTP
Peut mettre en cache les réponses pour réduire la charge sur les backends.

5. Terminaison TLS/SSL
Gère le HTTPS à sa place, et communique en HTTP simple avec les backends.

Pourquoi Nginx plutôt qu'Apache ?
Nginx a été conçu pour gérer un grand nombre de connexions simultanées avec peu de mémoire, 
grâce à une architecture événementielle asynchrone (non-bloquante), contrairement à Apache qui crée un thread/processus par connexion.

## Étape 4 — Installer Nginx

```bash
# Installer Nginx
apt install nginx -y

# Démarrer Nginx et l'activer au boot
systemctl enable nginx
systemctl start nginx
systemctl status nginx #Vérifier qu'il tourne
```

Tu devrais voir `active (running)` en vert.

Note l'IP de ton LXC :
```bash
ip a
```
Depuis ton navigateur sur le même réseau, tape cette IP — tu devrais voir la page par défaut de Nginx. ✅

## Étape 5 — Déposer tes fichiers HTML

Crée le dossier de ton site :

```bash
mkdir -p /var/www/monsite
```

Configure Nginx pour pointer vers ce dossier :

```bash
nano /etc/nginx/sites-available/monsite
```

Colle cette config :

server {
    listen 80;
    server_name mondomaine.com www.mondomaine.com;

    root /var/www/monsite;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }
}

Active la config :
# Créer le lien symbolique
ln -s /etc/nginx/sites-available/monsite /etc/nginx/sites-enabled/

# Supprimer le site par défaut
rm /etc/nginx/sites-enabled/default

# Tester la config (doit dire OK)
nginx -t

# Recharger Nginx
systemctl reload nginx
```

Pour déposer tes fichiers HTML depuis ton PC, utilise **SCP** ou **FileZilla** (SFTP) :

Pour autoriser la connexion SSH en tant que root, tu dois modifier le fichier de config du serveur SSH :
sudo nano /etc/ssh/sshd_config

Trouve la ligne PermitRootLogin et change sa valeur :
PermitRootLogin yes

Puis redémarre le service :

bash
sudo systemctl restart sshd

```bash
# Depuis ton PC, envoyer un fichier vers le LXC
scp index.html root@IP_DU_LXC:/var/www/monsite/
```

## Étape 6 — Ouvrir les ports sur ta box

C'est l'étape qui rend ton site accessible depuis Internet.

Tu dois faire de la **redirection de ports** (port forwarding) sur ta box :

| Port externe | Port interne | IP destination |
| 80 		    | 80 		   | IP de ton LXC |
| 443 	    | 443 		   | IP de ton LXC |

⚠️ **Problème possible** : ton LXC a une IP en DHCP, elle peut changer. Il faut lui donner une **IP fixe**.

Dans la console de ton LXC :

```bash
nano /etc/network/interfaces
```

Modifie comme ça :

```
auto eth0
iface eth0 inet static
    address 192.168.1.50      # choisis une IP libre chez toi
    netmask 255.255.255.0
    gateway 192.168.1.1       # IP de ta box
```

```bash
systemctl restart networking
```

Ensuite dans l'interface de ta box (tape `192.168.1.1` sur ton navigateur) cherche **NAT / Redirection de ports / Port Forwarding** et ajoute les règles du tableau ci-dessus.

## Étape 7 — Nom de domaine + SSL

**Tu as un nom de domaine ?** Si non, tu peux en acheter un pour ~10€/an chez OVH ou Namecheap.

Une fois que tu as ton domaine, dans la gestion DNS de ton registrar :

```
Type  : A
Nom   : @
Valeur: TON_IP_PUBLIQUE   ← cherche "mon ip" sur Google
```

```
Type  : A
Nom   : www
Valeur: TON_IP_PUBLIQUE
```

La propagation DNS prend 5 minutes à 24h.

**Ensuite, SSL gratuit avec Certbot :**

```bash
# Installer Certbot
apt install certbot python3-certbot-nginx -y

# Générer le certificat (remplace avec ton vrai domaine)
certbot --nginx -d mondomaine.com -d www.mondomaine.com

# Certbot configure HTTPS automatiquement et renouvelle seul
```

Ton site est maintenant accessible en **HTTPS** avec le cadenas vert. ✅
